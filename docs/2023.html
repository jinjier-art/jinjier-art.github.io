2023年度JAV行业报告

2024/03/15 发表于 jinjier.art


写在前面


2023年初时，我们曾通过数据分析的方式，对2022年度的JAV新人市场做过一个系统性的研究报告。

本次，我们仍将使用数据分析的方式，来对2023年度的整个JAV行业做一个系统性的研究报告。

注：本文支持移动设备，但因为内容较长，且包含大量图表📈，更建议使用电脑💻阅读。


数据来源


既然是数据分析，我们首先需要获取足够的数据。从数据来源上讲，与《2022年度新人报告》类似，主要是取自以下几个网站（部分需要翻墙访问）：


1. FANZA

https://www.dmm.co.jp

FANZA是JAV业界最大的正版门户网站。大部分JAV有码影片的信息都可以在FANZA上查到：包括了制作商、发行商、发行日期、导演、女优、类别等各种元数据，及影片的图片、视频预览等（大部分盗版网站的预览都是直接盗链FANZA）。

值得注意的是，FANZA同时包含通贩（实体碟片）和动画（流媒体）专区。我们的数据主要抓取自动画区，这是因为动画区的数据更全，包含了很多不在通贩区的流媒体影片；此外，动画区的UGC数据更多，包括了我们在下文中用到的「用户收藏数」这一重要指标。但通贩区也有一些我们需要的数据，如FANZA通贩销量榜这一重要的销量参考数据。

注：FANZA可能需要日本IP的VPN才能访问。


2. JavLibrary

https://www.javlibrary.com/cn/

JavLibrary主要收录JAV有码碟片信息，因此数据的很大部分都来自于FANZA通贩区。此外JavLibrary还收录了一些来自MGS、SOD等平台且不在FANZA发售的影片。

按照日本AV法规，业界演员有下架五年前作品的权利——因此FANZA等平台经常会下架一些历史影片——所以JavLibrary保存的历史数据会比FANZA更完整。此外JavLibrary的数据清洗也做得不错，一些改名的演员、伪素人等都会被重新链接。

除了影片元数据外，JavLibrary的UGC数据也极具参考价值，如：影片想看人数、看过人数，影片评分、评论，TOP500影片等。


3. JavDB

https://javdb.com

JavDB收录的影片理论上比FANZA和JavLibrary更全：不仅包括了有码影片，还包括无码、FC2、欧美、动漫等其他类型。但JavDB的数据清洗做得一般，大量影片存在数据错漏、关键信息丢失等问题。

除了影片元数据外，JavDB的UGC数据也极具参考价值，如：影片评分、评论，TOP250影片等。


4. MinnanoAV

https://www.minnano-av.com

2023年度片商评分排行


FANZA、JavLibrary、JavDB的影片页面都有用户评分功能。我们都可以获取到两个重要参数：评分，及参与评分的人数。从影片评分出发，我们进而可以统计出制作商的评分。

在进入片商评分之前，我们需要先引入一些概念和公式——不用紧张，不超过中学数学的范畴。

数学概念和公式：

  1. 算术平均值：
    R=∑i=1nrin
    R = \frac{\sum_{i=1}^{n} r_i}{n}
    R=n∑i=1n​ri​​
    算术平均值的计算很好理解，就是我们通常意义上的求平均值：将多个数相加，再除以个数。FANZA、JavLibrary、JavDB显示的影片评分即为用户评分的算术平均值。

  2. 加权平均值：
    Rˉ=∑i=1nrivi∑i=1nvi
    \bar{R} = \frac{\sum_{i=1}^{n}r_iv_i}{\sum_{i=1}^{n}v_i}
    Rˉ=∑i=1n​vi​∑i=1n​ri​vi​​
    我们用一个例子来解释加权平均值：假设有两部影片，第一部评分5.0，1人打分；第二部评分4.0，9人打分。则两部影片评分的算术平均值为5.0+4.02=4.5\frac{5.0+4.0}{2}=4.525.0+4.0​=4.5，而加权平均值为5.0×1+4.0×91+9=4.1\frac{5.0\times1 +4.0\times9}{1+9}=4.11+95.0×1+4.0×9​=4.1。我们在计算片商评分时，显然加权平均值比算术平均值更具参考性。
    注：换一个角度来看，用户评分本身没有权重，所谓的权重是由于不同影片评分人数不同导致的。因此片商的加权评分本质上是该片商的所有影片的所有用户评分的算术平均值。

  3. 贝叶斯加权平均值：
    RB=∑i=1nrivi+RV∑i=1nvi+V
    R_B = \frac{\sum_{i=1}^{n}r_iv_i+RV}{\sum_{i=1}^{n}v_i+V}
    RB​=∑i=1n​vi​+V∑i=1n​ri​vi​+RV​
    加权平均分并不能解决所有问题。我们来举一个比较极端的例子：片商A只发行了1部影片，只有1人评分5.0，则加权平均分也为5.0；片商B发行了100部影片，共10000人评分，加权平均分4.9。难道我们可以说片商A的评分高于片商B？显然不能。贝叶斯加权平均值一定程度上能解决这个问题。它的背后是贝叶斯公式思想的体现，IMDB、豆瓣、包括JavDB等网站的TOP250榜单都可以看作它的具体应用。

类别统计


所谓类别，即用来标识影片类型的标签。比如「首次亮相」标识着新人出道作；「女大学生」标识着女优来历，「4K」标识着影片分辨率。

一部影片可以有多个类别。如一部新人出道作，可以同时打上「首次亮相」、「单体作品」、「女大学生」、「4K」等多个标签。

不同数据来源的英文版对「类别」有不同的叫法。JavLibrary是genre（类型），JavDB是tag（标签），FANZA是keyword（关键词）。我们最终统一把它称为「类别」，是因为JavLibrary和JavDB的中文版都使用「类别」来指代它。

数据源特别说明

FANZA在存储影片的导演信息时和JavLibrary及JavDB存在着一些微妙的差别。以下说明对熟悉关系型数据库的朋友不难理解：
